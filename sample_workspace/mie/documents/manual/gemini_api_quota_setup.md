# Gemini API 1日利用上限の設定手順

**対象**: ひのちゃん（りんちゃんMac管理者）  
**目的**: Gemini APIの1日あたりの利用量に上限を設けて、予期せぬ課金を防ぐ

---

## 手順

### 1. Google Cloud Consoleにログイン

1. ブラウザで https://console.cloud.google.com/ を開く
2. りんちゃんのGemini APIキーを発行したGoogleアカウントでログイン

---

### 2. プロジェクトを選択

1. 画面上部のプロジェクト名をクリック
2. Gemini APIを有効にしているプロジェクトを選択

---

### 3. APIのクォータ設定画面を開く

1. 左メニューから **「APIとサービス」** → **「有効なAPI」** をクリック
2. 一覧から **「Generative Language API」**（またはGemini API）を探してクリック
3. 上部タブから **「クォータと上限」** をクリック

---

### 4. 1日の上限を設定する

1. クォータ一覧から設定したい項目を探す
   - `Generate content requests per day per project`（1日あたりのリクエスト数）
   - `Generate content input tokens per day`（1日あたりの入力トークン数）など
2. 設定したい項目の右端にある **鉛筆アイコン（編集）** をクリック
3. 上限値を入力して **「保存」**

---

### 5. 予算アラートも設定しておく（推奨）

クォータとは別に、**費用の上限アラート**も設定しておくと安心です。

1. 左メニューから **「お支払い」** → **「予算とアラート」** をクリック
2. **「予算を作成」** をクリック
3. 月の予算金額（例: ¥1,000）を設定
4. アラートのしきい値（50%、90%、100%など）とメール通知先を設定して保存

---

## 補足

- 無料枠の間はクォータ制限に引っかかることは少ないですが、設定しておくと有料移行後も安心です
- クォータを超えるとAPIが `429 Resource Exhausted` エラーを返してりんちゃんが応答できなくなります
- 上限は後からいつでも変更できます
